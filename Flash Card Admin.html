1. Administrative Procedures

What are the optional procedures for different Vitrea deployments?

Optional procedures for Enterprise, Enterprise Single Server, Vitrea extend, or Vitrea workstation deployments.

2. Windows DPI Settings

How do you adjust DPI settings for 4K displays?

Adjusts Dots Per Inch scaling for 4K Displays only. Performed for Vitrea extend or Vitrea workstation deployments. Set to 100%. Requires sign out to take effect.

3. Administrative Reveal Desktop

What does the Administrative Reveal Desktop feature do?

Allows Vitrea administrators to minimize the Vitrea application for administrative work. Available for Enterprise, Enterprise Single Server, Vitrea extend, and Vitrea workstation deployments. Launches Windows Explorer and reveals the Start menu. System resets on Kiosk session close.

4. Audit Logging

How does Vitrea AV handle user logons and audit logging?

Vitrea AV uses Integrated Windows Authentication for user logons. Audit logon events can be collected via Microsoft security audit policy settings.

5. Restarting Management Server (Enterprise)

What are the considerations for restarting the Management Server in an Enterprise deployment?

Ensure no Application Servers are restarting or have active user sessions in Vitrea. Restart the "VSP Application Service" on the Management Server separately after its services report online.

6. Restarting Application Servers (Enterprise)

What are the best practices for restarting Application Servers in an Enterprise deployment?

Avoid restarting while the Management Server is restarting. Ensure no active user sessions. Allow at least a minute between restarting multiple Application Servers.

7. CACS Report Guideline Default Setting

How is the CACS Report Guideline default setting determined?

Based on Vitrea software license locale; "Nasu" for Japanese, "Hoff" for United States. Available for all deployments.

8. Change AE Title (Upgrade)

What is required before upgrading to 7.16.3 regarding AE Title?

Required for all deployments before upgrading to 7.16.3. The AE Title must be changed to UPGRADE.

9. Default Export Destination

How do you configure the default export destination and what's required for changes to take effect?

Configurable for Enterprise, Enterprise Single Server, Vitrea Extend, and Vitrea workstation deployments. Active sessions must reconnect for changes to apply.

10. Evidence Order on Studylist Results

How is evidence ordered on Studylist Results by default and how can it be changed?

Default is "false," meaning the most recent snapshot is on the far right. Set newestToOldestResultOrdering to "true" for the most recent to display on the far left.

11. Change Hostname Applicability

When can you change the hostname of a Vitrea system?

Only for Enterprise Single Server, Vitrea extend, or a Vitrea workstation deployment. System must be removed from domain before changing hostname.

12. Hostname Restrictions

What are the restrictions for Vitrea hostnames?

The hostname cannot be "Vitrea". Limited to 15 characters.

13. VitreaHostnameTool.exe

What is VitreaHostnameTool.exe used for?

Used to update Vitrea configuration after changing hostname. A "Test Run" performs a dry run without applying changes.

14. Site Logo on Report Page

What are the requirements for the site logo on report pages?

Supports .JPG and .PNG files. Only applicable to Full Cardiac, Calcium Score, and Generic reports.

15. Change Storage Location (Enterprise Caution)

What should you consider when changing storage location in Enterprise deployments?

Copying data takes a significant amount of time. Canon Medical recommends starting copy well in advance.

16. NASRoot Directory UNC Path Limit

What is the UNC path length limitation for NASRoot directory?

The UNC path of the NASRoot directory must be 31 characters or fewer.

17. Robocopy Command

What command can be used to copy NASRoot contents?

Robocopy C:\SourcePath C:\DestinationPath /MIR /dcopy:T /SEC can be used to copy NASRoot contents.

18. SQL Server Database Location

What happens if you move a database to remote storage?

Moving a database residing on the Management Server to a remote storage device will affect performance.

19. Change Storage Location (Non-Enterprise Caution)

What should you know about changing storage location in non-Enterprise deployments?

This procedure is not meant to change storage to a network drive for Enterprise Single Server, Vitrea extend, or workstation deployments.

20. Configure AE Title for PrintSCU/QRSCU

What is required when configuring AE Title for PrintSCU/QRSCU?

Applicable to all deployments. Requires a unique AE Title value.

21. Autosave Feature (VitreaClassic)

What does the VitreaClassic autosave feature do?

Automatically creates a Vitrea Saved session upon idle timeout. Applies only to VitreaClassic, not other third-party applications. Disabled by default.

22. Remote Desktop Session Settings (Autosave)

What are the recommended Remote Desktop session settings for autosave?

"End a disconnected session" to 1 minute, "Active session limit" to Never, "Idle session limit" to 30 minutes. "Disconnect from the session" radio button should be selected.

23. CSMC Cardiac Suite Floating License Server Requirement

What is required for the CSMC Cardiac Suite Floating License Server?

Requires constant internet access (Call Home) to http://vm.csaim.com/. License will shut down if not reachable within 14 days.

24. CSMC License Acquisition

How do you acquire a CSMC license?

Contact Canon Medical Customer Support; typically requires 24-48 hours to receive the license file.

25. Vitrea Workstation CSMC Licensing

What are the licensing options for Vitrea Workstation CSMC?

Can be Local Node-Locked (installed on each workstation) or use a Floating License Server (not recommended to set up on a Workstation).

26. CT Colon CAD Keywords

What are the standard and custom keywords for CT Colon CAD?

Standard keywords are colon, prone, supine, and decubitus. Custom keywords are added using a | character.

27. CT Colon CAD Task Timeout

What is the default timeout for CT Colon CAD tasks?

Default value is 45 minutes. Vitrea retries processing up to three times after a timeout or load failure.

28. Deconvolution Methods (2D Brain Perfusion CT)

What deconvolution methods are supported for 2D Brain Perfusion CT?

Supported values are SVD, SVD+, and Bayesian.

29. Deconvolution Methods (4D Brain Perfusion CT)

What deconvolution methods are supported for 4D Brain Perfusion CT?

Supported values are SVD+ and Bayesian.

30. Evidence Auto Export Configuration

How do you configure evidence auto export?

Set performAutoExport to "true" and autoExportServerKeys with desired archive keys (comma-separated).

31. "report" EvidenceType (Auto Export)

What happens if you enable "report" EvidenceType for auto export?

If enabled, you lose the ability to create a draft report. Report is committed and automatically exported upon saving/publishing.

32. Export Reminder Feature

What does the export reminder feature do?

Displays a dialog reminding the user to export findings/reports after closing all applications for a study. Default is "off" (false).

33. External Groups (Mapping)

How are external groups mapped in Vitrea?

Maps Active Directory group(s) to Vitrea roles (Service, User, Admin). Never deselect the Admin Role for the BUILTIN Administrators Group.

34. Global Illumination Batch/Movie Quality

What are the quality settings for Global Illumination rendering?

Default is Medium. Values can be "Low," "Medium," or "High". Lower quality for faster playback.

35. Increased Output Size for Published Reports

What happens when you enable resizeDropAreas for published reports?

Enabling resizeDropAreas increases report image file size by 2.5x.

36. Interpreting Physicians and Operators (Ultrasound)

How are interpreting physicians and operators managed for ultrasound?

Manually added/edited/deleted within System Configuration > Ultrasound. Linked to the Vitrea Study List.

37. Just-In-Time (JIT) Processing

What is JIT Processing in Vitrea?

Enables/disables and configures algorithms that run within the JIT framework. Algorithms are processed when system resources are available.

38. Mirada RTX Evidence Blacklisting

How do you prevent evidence blacklisting in Mirada RTX?

Change export.sc.compatibility to false in XD.cfg to prevent blacklisting.

39. Open Rib JIT Mode (OpenRibAP)

What does OpenRibAP JIT mode do?

Runs Rib segmentation in the background to cache results for faster load times. Only processes CT Datasets and ORIGINAL PRIMARY images.

40. OpenRibAP Filter Keyword

What is the default filter keyword for OpenRibAP?

Default filter is "openrib" in the Series/Study Description.

41. Remote Desktop Sessions (Rejoining)

What is the default behavior for rejoining Remote Desktop sessions?

Default prevents rejoining sessions from different client systems. Setting AllowDuplicateUserConnections to "true" allows rejoining but may result in undefined behavior.

42. Series Sequence Numbering Defaults

What are the default series sequence numbering values?

initialSequenceNumber is 500, incrementalStep is 1.

43. SQL Server Permissions (Vitrea Administrators)

What permissions are required for Vitrea Administrators in SQL Server?

The Vitrea Administrators group must exist. Grant sysadmin role to this group.

44. SQL Server Services Recovery

How should SQL Server services be configured for recovery?

Ensure SQL Server (VITALSQLEXPRESS) is set to Restart the Service on first, second, and subsequent failures.

45. TAVR Systole/Diastole Ranges

What are the default TAVR systole and diastole ranges?

Default systole range is 30-40%, diastole range is 65-75%.

46. Uncommitted Evidence Retention Period

What is the default retention period for uncommitted evidence?

Default changed to 1000 years (from 6 months) for Vitrea 7.10+. Configurable in ISO-8601 format.

47. Vitrea Lung CAD Availability

Where is Vitrea Lung CAD available?

Only available in the OUS (Outside United States). Requires a new CT Vitrea Lung CAD license.

48. 3D Angio XA Archive Types

How are 3D Angio XA archives configured?

Must be configured to use either Toshiba or Standard positioner angles. Creating two archives with different keys allows exporting both c-arm angles and entire studies.

49. Database Backup Job

When are database backups scheduled?

Scheduled daily at 1:00 a.m. (local time). Keeps five copies of backup files.

50. Database Restore Version Compatibility

What version requirements exist for database restore?

The database schema is coupled to the build version; you must be running the same version of Vitrea software as when the backup was created.

Troubleshooting Scenarios

Showing 25 of 25 scenarios

1. DPI Settings Not Taking Effect

You are adjusting Windows DPI settings for a Vitrea extend deployment on a 4K display, but the changes do not seem to take effect.

Resolution: You must sign out after changing the setting for the new DPI scaling factor to take effect.

2. Application Servers Going Offline During Management Server Restart

While restarting the Management Server in an Enterprise deployment, Application Servers suddenly go offline on the System Health page.

Resolution: Ensure there are no Application Servers in the middle of being restarted when restarting the Management Server. Also, avoid restarting Application Servers while the Management Server is restarting.

3. CACS Report Guideline Changes Not Reflecting

You are attempting to change the CACS Report Guideline default setting, but the changes are not immediately reflected in the Vitrea software.

Resolution: After changing the guideline, the new default value should be selected automatically in the Vitrea software. If not, ensure you clicked "Activate Changes" and performed the steps correctly.

4. AE Title Change Before Upgrade

An upgrade to Vitrea 7.16.3 is planned, but patient data processing is incomplete. You're told to change the AE Title for the upgrade but aren't sure which one.

Resolution: You must change the AE Title to "UPGRADE" to ensure all patient data has been processed before the upgrade.

5. Export Destination Changes Not Visible to Users

You've changed the default export destination in Vitrea AV, but users currently working in active sessions are not seeing the new default.

Resolution: All active sessions during the change must reconnect to Vitrea before the new default archive will show as selected.

6. Hostname Change Causes System Failure

You change the hostname of an Enterprise Single Server after Vitrea installation, but Vitrea no longer functions.

Resolution: After changing the hostname, Vitrea will no longer function and requires specific post-hostname change procedures, including running VitreaHostnameTool.exe and potentially reinstalling VIMS components if the installation screen displays.

7. HostnameUpdate Tool Fails

You are attempting to change the hostname of your Vitrea system, but the HostnameUpdate Tool fails.

Resolution: The HostnameUpdate Tool will fail if the hostname is "Vitrea". Do not use "Vitrea" as the hostname.

8. Site Logo Not Displaying Correctly

You changed the site logo on the Report page by placing a new image file, but it doesn't display correctly, or is not available for all reports.

Resolution: Ensure the logo image is in .JPG or .PNG format and renamed to logo.gif. The change only applies to Full Cardiac, Calcium Score, and Generic reports.

9. NASRoot Path Length Concern

You're changing the storage location for an Enterprise deployment, and you're concerned about the path length.

Resolution: The UNC path of the NASRoot directory must be 31 characters or fewer due to Windows max path length limits.

10. SQL Server Performance Issues After Storage Change

After changing the NASRoot directory, you encounter issues with SQL Server database performance.

Resolution: If your database resides on the Management Server, moving it to a remote storage device will affect your performance. Confirm and update database backup locations if the database location is changed.

11. CSMC License Server Activation Failure

You are setting up a CSMC Cardiac Suite Floating License Server, but activation fails with an error message indicating a lack of internet access.

Resolution: The CSMC Cardiac Suite Floating License Server requires a Call Home (constant internet access) to http://vm.csaim.com/. You may need to set up rules in your proxy server to allow this connection or whitelist the site.

12. CSMC License Server Stops Functioning

The CSMC Cardiac Suite Floating License Server stops functioning after two weeks, and you did not provide constant internet access.

Resolution: The Floating License Server performs a Call Home every day for two weeks and then shuts down if constant internet access is not provided. To reset it, you need to obtain a reset code via email from http://:6433 and enter it into the Reset Code field.

13. CT Colon CAD Not Processing

You enable CT Colon CAD processing, but studies with relevant keywords are not being processed automatically, and you see "No License" in the Status column for CTColonCADJIT.

Resolution: This indicates a Colon CAD license does not exist on the system. You need to add a valid Colon CAD license for JIT processing to enable and create CADProcess.WorkItem.

14. StudyList Status Icon Changes

After enabling an intensive task like Colon CAD, you notice the StudyList status icon changes from yellow to green, then back to yellow, or disappears entirely for a study that should be fully processed.

Resolution: This can occur when multiple intensive tasks (e.g., Colon CAD and Lung CAD) are triggered for a given study. One algorithm finishes, and another starts. You can manually request Colon CAD or Lung CAD via the StudyList context menu or launch an appropriate application (like CT Colon Analysis) to trigger the algorithms if they haven't run or failed.

15. Evidence Not Auto-Exporting

You have configured Evidence Auto Export, but some evidence types are not being automatically exported.

Resolution: Check the supportedEvidenceTypes key in EvidenceAutoExporter. By default, it includes image, batch, and session. If you want to auto-export reports, you must add report to this comma-separated list.

16. Report Printing Margin Issues

You are printing a report from the Report page, and the margins appear too large, cutting off content.

Resolution: This issue can be addressed by editing the VI_report_print.css file located at C:\Program Files\Vital Images\Reporting\web\static\Javascript and adjusting the desired page margin.

17. Large File Sizes After Enabling resizeDropAreas

You've enabled resizeDropAreas to improve readability of snapshot overlay text in published reports, but you notice a significant increase in file sizes for your reports.

Resolution: This is expected behavior. When resizeDropAreas is enabled, report images will increase in file size by 2.5x.

18. Monitor Preference Option Unavailable

You're logged into the Vitrea software, but when you attempt to select "Set Monitor Preference" from the gear icon, the option is unavailable.

Resolution: This option is only available when not in an active session. Once the monitor preference is set, users are prevented from changing the monitor settings during a session. You need to relaunch Vitrea to reset it.

19. SUREPlaque Measurements Not Exporting

You're trying to view SUREPlaque measurements in a spreadsheet, but no .csv file is generated when you press Alt+v in the volume view.

Resolution: Ensure you pressed v after the cursor shape changed to the hand holding two arrows (by holding down Alt). Also, verify that the three required registry keys (VesselOutputType, VPSpreadSheetFormat, VesselFileLocation) are correctly set up.

20. VSP Services Fail to Start

VSP services on your Vitrea workstation deployment fail to start, and no logs are generated.

Resolution: Check the Event Viewer for errors. Verify that the service user is granted "Logon as Service" rights to the local server, and check if the service user's password has changed.

21. Database Restore Version Mismatch

You need to restore a database backup, but the Vitrea software version on your system is newer than the version when the backup was created.

Resolution: The database schema is coupled to the build version. You must be running the same version of the Vitrea software as was installed when the database backup was created to successfully restore it.

22. Studies Stuck in VIMS Processing

Studies become "stuck" in VIMS processing and are not visible in Vitrea, even though new studies are processing correctly.

Resolution: For Vitrea 7.12 and later, you can resolve this by: 1) Stopping all VIMS services; 2) Deleting entries in Dicom.ReceivedStudy_Temp table in the VIDicom database; 3) Deleting everything in the DICOMReceive folder; 4) Restarting VIMS Services; 5) Refreshing the Study List and manually deleting the "stuck" studies.

23. DICOM Processing "Sawtoothing" Graph

You're monitoring DICOM processing using VIMS Monitor, and the "Remaining Instances" graph displays a "sawtoothing" line instead of a flat line at zero.

Resolution: A "sawtoothing" line indicates there is still incoming data being processed. You should wait for all incoming data to complete processing before assuming a problem.

24. TAVR Iliac Measurements Not Automatic

You enabled the TAVR_Iliac_Access_Initialized setting, expecting automatic Iliac measurements, but users still need to define them manually.

Resolution: The default is TRUE for automatic measurements. To disable automatic Iliac access measurements and force manual definition, you need to set TAVR_Iliac_Access_Initialized to false.

25. Rejoining Remote Desktop Sessions

You want to allow a user to attach to an existing remote desktop session on an Enterprise deployment from a different client system, but the default configuration prevents this.

Resolution: Create a RemoteDesktopInfo.Service.exe.config file in C:\ProgramData\Vitrea\DeploymentConfiguration and set the AllowDuplicateUserConnections value to true. Be aware that this may result in undefined behavior.

Quick Tips and Summary

Quick Tips

  • Deployment Specificity: Always check the "N O T E" section in the documentation; procedures often apply only to specific deployment types (Enterprise, Enterprise Single Server, Vitrea extend, or Vitrea workstation).
  • Sign Out/Restart: Many configuration changes, especially those affecting Windows settings (like DPI) or application-level settings, require you to sign out of Vitrea or restart the software/machine for changes to take effect.
  • AE Titles: When configuring DICOM devices or changing hostnames, always use unique AE Title values where specified.
  • Licensing: For specialized features like CSMC Cardiac Suite or CT/Lung CAD, a valid license is required. Contact Canon Medical Customer Support for license acquisition. Be aware of "Call Home" requirements for floating licenses.
  • System Configuration: The "System Configuration" screen in Vitrea (accessed via Applications Home tab) is your central hub for most administrative settings.
  • Intensive Tasks (JIT): Be mindful that applications like Colon CAD and Lung CAD are "Intensive tasks" that require significant system resources. Monitor their status via the "Work Items" screen.
  • Data Migration/Storage: Copying large amounts of data for storage location changes can take a significant amount of time. Ensure UNC paths adhere to character limits (e.g., 31 characters for NASRoot).
  • Database Management: Regular database backups are scheduled automatically. Manual backups are crucial before upgrades. For database restoration, the Vitrea software version must match the version when the backup was created.
  • Registry Keys: Many advanced configurations or troubleshooting steps involve modifying Registry Keys (e.g., changing NASRoot, Journaling, SUREPlaque measurements). Exercise caution and follow instructions precisely.
  • System Health: Use the "System Health" application to monitor the status of services, certificates, user sessions, and hardware. Pay attention to "Immediate Action Required" alerts for certificates.

Summary

Vitrea administration encompasses a wide range of tasks, from basic Windows settings adjustments for 4K displays to complex server and database configurations for Enterprise deployments. Key administrative areas include managing server restarts and services, configuring DICOM-related settings like AE Titles for various functionalities (upgrade, print, query/retrieve), and customizing data handling such as export destinations and evidence retention periods.

The system offers "Just-In-Time Processing" for automated analysis applications like CT Colon CAD and Vitrea Lung CAD, which can be configured based on specific DICOM tags and keywords. User and group management, including mapping Active Directory groups to Vitrea roles, is handled through the "Group / Role Mapping" section. Performance can be optimized for specific applications like Olea Breast, and diagnostic tools like "Journaling" help record and play back workflows for troubleshooting. Critical system health and data processing can be monitored using the "System Health" and "VIMS Monitor" applications, and procedures exist for resolving "stuck" studies.

Overall, successful Vitrea administration relies on careful adherence to documented procedures, understanding deployment-specific requirements, and proactive monitoring of system health and processes.

Deep Dive: Changing the Storage Location (NASRoot Directory)

Enterprise Deployments: Typically undertaken to expand storage capacity or relocate the NASRoot directory from the Management Server to a dedicated remote server for better scalability and performance. This is a time-consuming process if existing data needs to be copied, potentially taking days for multiple terabytes of data.

Key Steps:

  1. Preparation: Add new storage or set up the external storage device share with appropriate permissions.
  2. Service Stoppage: All Vitrea services on ALL Vitrea Servers must be stopped using a PowerShell script.
  3. Database Handling: If the database resides within the NASRoot, it must be detached from SQL Server Management Studio before data copying.
  4. Data Copying & Path Limit: The UNC path of the new directory must be 31 characters or fewer due to Windows path length limits.
  5. Permissions: Meticulous configuration of Network Share Permissions and NTFS Directory Permissions is paramount.
  6. Database Re-attachment & Updates: The database must be attached to the new location and configuration updated.
  7. System Configuration Updates: The new NASRoot location must be updated in numerous Registry Keys and configuration files.
  8. Service Re-enablement & Verification: All Vitrea services must be re-enabled and started.

Enterprise Single Server, Vitrea extend, and Vitrea workstation Deployments: This procedure is specifically for correcting a misconfiguration where the VitreaData folder was accidentally placed on an incorrect local drive. It is NOT intended to change the storage location to a network drive for these deployments.
