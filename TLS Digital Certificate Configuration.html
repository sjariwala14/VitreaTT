Certificate Requirements

Based on your deployment scenario, select the appropriate certificate type below. This interactive guide will show you the specific requirements for each scenario.

Single Server Deployment

For a single server deployment with on-box SQL Server, you need one digital certificate with these properties:

Digital Certificate
server1.yourdomain.com
Not required
AT_KEYEXCHANGE (if SQL 2017)

Configuration Checklist:

  • Certificate must be SHA256 or better
  • 2048-bit RSA key or 256-bit ECC key minimum
  • Supports TLS 1.2
  • Valid for 1 year
  • Issued by internal CA or trusted third party

SAN Certificate Deployment

Wildcard Certificate Deployment

For large deployments or unlimited hosts, a wildcard certificate can simplify management:

Wildcard Certificate
*.yourdomain.com
Not applicable
Cannot be used for SQL 2017

Note: Wildcard certificates currently cannot be used for SQL Server 2017. For SQL 2019 or higher, the same wildcard certificate can be used for SQL.

Deployment Examples:

  • Scenario One: 1 Management Server, remote SQL Server, and 10 Application Servers - one wildcard certificate
  • Scenario Two: Large deployment with 20+ servers - wildcard simplifies certificate management
  • Scenario Three: Multi-domain environment - requires separate wildcards or SAN certificates

IIS Certificate Binding

After upgrading Vitrea, you may need to re-establish IIS certificates and SSL bindings:

Why is this needed? Upgrading Vitrea can cause IIS certificates and SSL bindings to be removed during the upgrade process.

Steps to verify IIS bindings:

  1. Open Server Manager
  2. Navigate to Roles â†’ Web Server â†’ IIS Manager

Pro Tip: Save a copy of your digital certificate to a safe location before upgrading, then remove it after completing the upgrade.
