Troubleshooting Guide

Users get "No sessions available" with 4 connections

Simple

Root Cause: Exhausted RDS CALs or service crash

Steps:

  1. Check sessions: qwinsta /server:localhost
  2. Verify RDS CALs in Remote Desktop Licensing Manager
  3. Restart VitreaRemoteDesktopServices service

Reference: Page 66

DICOM Transfer Failure: "Association Rejected"

Medium

Root Cause: AE Title mismatch or firewall issue

Steps:

  1. Confirm AE Title/Port in Vitrea Administration Tool â†’ DICOM Receive
  2. Test connection: .\echoscu.exe -aet VIMS_AE -aec PACS_AE 192.168.1.100 104
  3. Check firewall for port 3002

Fix: Update AE Title in DICOMReceive.xml and restart VI.Enterprise.Servers.Acceptor

Reference: Page 150

SQL Deadlocks During Peak Hours

Hard

Root Cause: Lock contention

Diagnostic Command:

SELECT * FROM sys.dm_tran_locks WHERE request_status='WAIT'

Solution: Set isolation level to READ UNCOMMITTED or add NOLOCK hints (Page 49)

Slow Rendering with 1K Datasets

Hard

Symptoms: Lagging volume rendering with 1024x1024 matrices; GPU at 100%.

Steps:

  1. Verify NVIDIA driver (553.62, Page 27)
  2. Check VitreaAvSlotSize registry: HKEY_LOCAL_MACHINE\SOFTWARE\Vital Images\Vitrea
  3. Confirm page file at 100GB on E: drive (Page 180)

Fix:

  • Downgrade to 512x512 datasets or
  • Upgrade to current-generation GPU (e.g., NVIDIA RTX 5000 Ada or equivalent)
  • Consider distributed rendering for large datasets

Reference: Page 180

AutoDelete Purges Active Studies

Aggressive

Root Cause: Misconfigured disk-based marking (Page 56)

Steps:

  1. Stop AutoDelete: Stop-Service -Name "VI.VIMS.Services.AutoDelete"
  2. Recover from E:\DICOMReceive\DICOMInstanceData
  3. Set MinimumDiskLimit to 50GB

Reference: Page 54

Ransomware Encrypts E:\VitreaData

Aggressive

Root Cause: Exploited vulnerability (e.g., CVE-2024-1234) or phishing attack

Immediate Action:

Set-SmbServerConfiguration -EnableSMB1Protocol $false
Disable-NetFirewallRule -DisplayGroup "SMBv1"

Recovery:

  1. Isolate server from network immediately
  2. Identify patient data impact through logs
  3. Restore from offline backups (maintained separately from network)
  4. Apply all current security patches before bringing back online

Prevention:

  • Enable system hardening script during installation
  • Implement regular offline backups
  • Keep all systems updated with latest security patches

Reference: Vitrea Security Best Practices, 2025 Edition

Integrated Windows Authentication (IWA) Failure

Medium

Symptoms: Users cannot log in despite correct AD credentials; authentication prompts repeatedly appear.

Possible Causes:

  • SPN (Service Principal Name) not configured properly
  • Kerberos ticket issues
  • IWA not enabled in Vitrea Administration Tool

Troubleshooting Steps:

  1. Verify IWA is enabled in Vitrea Administration Tool â†’ Authentication
  2. Check SPN configuration: setspn -L VITREA-SERVER$
  3. Test Kerberos ticket generation: klist get HTTP/vitrea-server.domain.com
  4. Verify time synchronization between Vitrea server and domain controllers

Reference: Vitrea Domain Deployment Guide, Chapter 5

Troubleshooting Flowcharts

DICOM Transfer Failures

Key Steps:

  1. Verify AE Title and port configuration in Vitrea Administration Tool
  2. Test connectivity: Test-NetConnection PACS_IP -Port 3002
  3. Check firewall rules for ports 3002 (STORE) and 3003 (QUERY/RETRIEVE)
  4. Restart VI.Enterprise.Servers.Acceptor service
  5. Verify PACS is configured to accept connections from Vitrea AE Title

Reference: Vitrea DICOM Configuration Guide, Page 150

SQL Deadlock Resolution

Key Steps:

  1. Detect deadlock
  2. Check isolation level (Read Committed or Repeatable Read)
  3. Optimize queries or add NOLOCK hints
  4. Verify: SELECT * FROM sys.dm_tran_locks WHERE request_status='WAIT'

  4. Restore from clean backups

Definitions:

  • EMR (Electronic Medical Record): Electronic record of health-related information on an individual, aiming to replace paper medical records.
  • PHR (Personal Health Record): Patient-maintained health records, providing a complete summary accessible online with credentials.
  • DICOM (Digital Imaging and Communications in Medicine): Primary standard for medical imaging, ensuring interoperability since 1985.
  • SCU/SCP: Service Class User (request side) and Service Class Provider (service side) in DICOM communication.
  • MPR (Multi-Planar Reformat): Axial, coronal, sagittal images reconstructed from a single-plane study in Vitrea AV.

User Roles:

  • Oncologist: Physician with advanced schooling in cancer.
  • Orthopedic Surgeon: Physician with advanced schooling in muscles, bones, joints, and spine.

  • Nuclear Medicine Technologist: Acquires PET scans or Nuclear Medicine studies.
  • PACS Administrator: Monitors images in/out of PACS, often with IT or technologist background.
  • IT Professional: Has networking or computer science background, sometimes also a PACS Administrator.
