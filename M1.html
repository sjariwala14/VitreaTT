I. General System Configuration
Windows DPI Settings (4K Displays)
Recommended setting: 100%
Sign out required for changes
Applicable to: Vitrea extend/workstation deployments
For optimal display quality on high-resolution monitors
Administrative Reveal Desktop
Minimizes Vitrea to perform admin work
Launches Windows Explorer
System resets to defaults when Kiosk session closes
Access through Vitrea System Settings
Select "Reveal Desktop" option
System Settings
Central configuration section
Select "Make Changes"
Enter reason for change
Click "OK" then "Activate Changes"
Many settings require Vitrea restart
Key configuration areas:
- Application behavior
- DICOM settings
- User interface preferences
Restarting Servers and Services
Enterprise Deployment:
Ensure no Application Servers are restarting
Verify no active user sessions
Stop Management Server services in correct order
Single Server/Extend/Workstation:
Stop all VSP services
Perform maintenance
Restart services in reverse order
Troubleshooting steps if services don't start:
Check Event Viewer for errors
Verify service user has Logon as Service rights
Confirm password hasn't changed
Key services to monitor:
- VSP Management Service
- VSP Application Service
- VSP Database Service

II. User & Permission Management
User Access Overview
Supported authentication methods:
- Windows groups (Active Directory or local)
- Local database users
Role-based permissions structure:
- Service: System-level access
- User: Standard application access
- Admin: Full administrative privileges
-- Example SQL query to view role mappings SELECT g.GroupName, r.RoleName FROM UserGroups g JOIN GroupRoles gr ON g.GroupID = gr.GroupID JOIN Roles r ON gr.RoleID = r.RoleID
External Groups (AD)
Navigate to User Management in Vitrea Admin Console
Select "Add External Group"
Enter AD group name (domain\group)
Assign appropriate roles (Service, User, Admin)
Critical security note:
- Never deselect Admin Role for BUILTIN\Administrators
- This would lock all administrators out of Local Vitrea
Registry keys involved:
- HKLM\Software\Vital\Vitrea\Security\ADGroups
Local Database Groups
Applicable to: Single Server, Extend, Workstation
Access Local User Management
Create new group
Assign permissions from template or custom
Add users to group
-- SQL to create local group (example) INSERT INTO LocalGroups (GroupName, Description) VALUES ('Radiologists', 'Group for radiology staff')
Local groups are stored in Vitrea database tables LocalGroups and LocalGroupMembers
Audit Logging
Uses Integrated Windows Authentication
Configure Microsoft security audit policy:
1. Open Local Security Policy (secpol.msc)
2. Navigate to Security Settings â†’ Local Policies â†’ Audit Policy
3. Enable "Audit logon events" and "Audit object access"
Key audit events tracked:
- User logons/logoffs
- Configuration changes
- Sensitive data access
-- Query audit logs example SELECT * FROM AuditLog WHERE EventTime > DATEADD(day, -7, GETDATE()) ORDER BY EventTime DESC
Single Sign-On (SSO)
Controlled by registry key:
- HKLM\Software\Vital\Vitrea\Security\singleSignOnEnabled
Default values:
- true (Workstation/Enterprise)
- false (Extend in kiosk mode)
To disable SSO:
1. Open Registry Editor
2. Navigate to the key above
3. Change value to 0
4. Restart Vitrea services
Permission assignment:
- Windows Administrators group â†’ Vitrea Admin role
- Windows Users group â†’ Vitrea User role
III. Data Management
Changing Storage Location (NASRoot)
Enterprise Deployment:
This is a complex, time-consuming procedure
1. Stop all Vitrea services on all servers
2. Detach database using SQL Management Studio
3. Copy data using robocopy:
robocopy \\oldnas\share \\newnas\share /MIR /COPYALL /R:1 /W:1
4. Set NTFS permissions (Full Control for VSP service account)
5. Update SQL tables:
UPDATE SystemSettings SET Value='\\newnas\share' WHERE Key='NASRoot'
6. Update registry keys:
- HKLM\Software\Vital\VSP\DataPath
7. Update config files:
- E:\VitreaData\Config\vsp.config
8. Reattach database
9. Restart all services
UNC path must be â‰¤31 characters due to Windows limitations
Moving SQL database to remote device will significantly impact performance
Single Server/Extend/Workstation:
1. Stop all VSP services
2. Move VitreaData folder to new location
3. Update registry keys:
- HKLM\Software\Vital\VSP\DataPath
4. Update config files:
- [InstallDir]\Config\vsp.config
5. Restart services
Not recommended for changing to network drive
Database Backup/Restore
Scheduled backup job runs daily at 1:00 a.m.
Retention: five copies of backup files
Default locations:
- Single/Extend/Workstation: E:\VitreaData\DBBackup
- Enterprise: \\[machinename]\nasroot\DBBackup
Manual backup procedure:
1. Open SQL Management Studio
2. Right-click Vitrea database â†’ Tasks â†’ Back Up
3. Select backup type (Full)
4. Choose destination
5. Execute backup
Restore procedure:
Must be running same Vitrea version as backup
1. Stop all Vitrea services
2. Open SQL Management Studio
3. Right-click Databases â†’ Restore Database
4. Select backup file
5. Check "Overwrite the existing database"
6. Execute restore
7. Restart services
-- Check database version compatibility SELECT [Version] FROM DatabaseSchema WHERE SchemaID = (SELECT MAX(SchemaID) FROM DatabaseSchema)
Evidence Auto Export
Configuration steps:
1. Navigate to System Settings â†’ Evidence
2. Select "Enable Auto Export"
3. Add DICOM archive destinations
4. Configure Evidence Types:
- Images
- Batches
- Sessions
- Reports (disables draft functionality)
Registry keys involved:
- HKLM\Software\Vital\Vitrea\Export\AutoExportEnabled
- HKLM\Software\Vital\Vitrea\Export\ArchiveKeys
-- SQL to verify configured archives SELECT ArchiveKey, AE_Title, Host, Port FROM DICOMArchives WHERE AutoExport = 1
Uncommitted Evidence Retention
Default: 1000 years (Vitrea 7.10+)
Previous default: 6 months
To modify retention period:
1. Open Registry Editor
2. Navigate to HKLM\Software\Vital\Vitrea\Evidence
3. Create/modify DWORD RetentionPeriodDays
4. Set value in days (365000 = ~1000 years)
5. Restart Vitrea services
Setting to extremely high values may cause disk space issues
-- SQL to check current retention settings SELECT EvidenceType, RetentionDays FROM EvidenceRetentionSettings
Max Data Saved to Media
Controlled by setting:
- HKLM\Software\Vital\Vitrea\Export\maxSaveToMediaSize
Default: 49GB (recommended)
Modification procedure:
1. Open Registry Editor
2. Navigate to the key above
3. Modify value (in bytes)
4. Restart Vitrea services
Higher values may:
- Stress system resources
- Cause longer processing times
- Risk timeouts during export
