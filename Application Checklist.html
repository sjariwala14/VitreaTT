Troubleshooting Checklist

1. Installation Fails or Aborts

Symptoms:
  • Installation executable crashes or terminates unexpectedly
  • Error messages such as "Installation failed" or "Prerequisites not met"
  • System reboots unexpectedly during installation
Resolution Steps:
  1. Review Installation Logs: Check log files in the installation directory (e.g., %ProgramFiles%\Application\Logs\Install.log or similar)
  2. Verify Prerequisites: Confirm .NET Framework version (e.g., 4.8 or newer) is installed
  3. Check Disk Space: Ensure sufficient free disk space (e.g., 400GB+ for application data and binaries)
  4. Disable Antivirus Temporarily: Pause antivirus software during installation
  5. Run Installer as Administrator: Right-click the installer and select "Run as administrator"

Reference: VPMC-17427B Pages 15-26 (Installation Process)

2. Application Service Fails to Start

Symptoms:
  • Service does not start after installation
  • Event Log errors like "Service terminated unexpectedly" or Error 1053
  • Ports required by the application are not listening
Resolution Steps:
  1. Verify Service Account: Ensure the service account has not expired and has "Log on as a service" rights
  2. Check Dependencies: Confirm SQL Server connectivity and Windows Firewall settings
  3. Review Event Logs: Open Event Viewer and filter for application-specific events
  4. Restart Service: Use services.msc to restart the application service

Reference: VPMC-17427B Pages 119-124 (Windows Firewall), 206-207 (SQL Permissions)

3. Database Connectivity Issues

Symptoms:
  • Errors like "Database deployment failed" or "Cannot connect to SQL Server"
  • Slow performance when accessing application data
  • Timeout errors in application logs
Resolution Steps:
  1. Verify SQL Server Configuration: Confirm SQL Server is running with sufficient resources
  2. Check Network Connectivity: Test connectivity to SQL Server
  3. Validate Permissions: Ensure the application service account has appropriate roles
  4. Optimize SQL Performance: Enable "Lock Pages in Memory" and set Max Degree of Parallelism

Reference: VPMC-17427B Pages 27-35 (Database Deployment), 206-207 (SQL Permissions)

4. Network and DICOM Connectivity Issues

Symptoms:
  • DICOM C-ECHO tests fail to PACS or modalities
  • Studies not appearing in the application
  • "DICOM Association Rejected" errors in logs
Resolution Steps:
  1. Verify AE Titles: Ensure Application Entity Titles are unique and alphanumeric
  2. Check Network Connectivity: Ping PACS/modality IPs from the server
  3. Validate Firewall Rules: Ensure inbound rules exist for DICOM ports
  4. Check DICOM Configuration: Verify DICOM device settings in System Configuration

Reference: VPMC-17427B Pages 60-67, 110-112 (DICOM Settings)

5. Licensing Activation Failures

Symptoms:
  • "Invalid license file" or "Host ID mismatch" errors
  • Application features disabled after a trial period
  • License application fails in the management interface
Resolution Steps:
  1. Verify Host ID: Compare HostInformation.xml with the license file
  2. Check License File: Ensure the license file is saved in the correct folder
  3. Apply License: Open the application's licensing interface and apply the license file
  4. Contact Support: Email vendor support with HostInformation.xml and error details

Reference: VPMC-17427B Pages 12-14, 55-56 (Licensing)

6. SSL/TLS Certificate Issues

Symptoms:
  • Browser warnings like "Invalid certificate" when accessing web components
  • Remote Desktop or web application connections fail with SSL errors
  • Application components not loading due to certificate issues
Resolution Steps:
  1. Verify Certificate: Check the certificate's expiration date and SAN
  2. Check Permissions: Grant NETWORK SERVICE read access to the certificate's private key
  3. Validate IIS Bindings: Ensure HTTPS bindings use the correct certificate and port
  4. Replace Certificate if Needed: Install a new customer-provided SSL/TLS certificate

Reference: VPMC-17427B Pages 119-121, 197-201 (SSL/TLS Configuration)
