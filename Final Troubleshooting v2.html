Applications Not Launching

Applies to: Enterprise, Enterprise Single Server, Vitrea Extend deployments

Possible Causes:

  • Licensing issues (confirm feature is purchased and available)
  • Application failing to get required resources
  • Internal application error
  • Service issues (v7.14.2 specific fix: restart services)
  • v7.14.2: Application icons not displaying, unable to open studies (often resolved by restarting services)

Solution:

  • Check licensing status at: http://localhost:8700/management/api/license/usage
  • For “Application name (process id #) terminated abnormally”: Collect the application’s logfile and exception info.
  • For “Vitrea Client has encountered an error and cannot retrieve required study information. GSB Error Code: ##”: Collect logs for AppShell and SessionHelper.
  • For “Unable to acquire system resources...”: Ensure correct NVIDIA driver and close other user sessions.
  • v7.14.2 Scaling Errors:
    1. Right-click Vitrea shortcut
    2. Properties → Compatibility
    3. Change high DPI settings
    4. Override scaling behavior → SYSTEM
  • v7.14.2 Icon/Study Issues: Restart Vitrea services using: 
    wmic service where "name like 'Vitrea.%%'" stopservice
wmic service where "name like 'Vitrea.%%'" startservice

StudyList Displays but No Data

Applies to: Enterprise, Enterprise Single Server, Vitrea Extend, Vitrea Workstation deployments

Troubleshooting Steps:

  • Check UI filters (set Study Date to “Any”)
  • Click Search to force refresh
  • Verify VAT data directory location
  • Check AutoDelete settings vs. disk space
  • Review logs: C:\ProgramData\Vitrea\Logs\[VIMS, Management, StackDefinitionBuilder, StudyList, ThumbnailService]

Understanding Study Ingestion

Process Overview:

  1. DICOM files transferred to Vitrea
  2. Vitrea builds DAR file
  3. Study stored in database
  4. Appears on StudyList
  5. Application stacks built

Key Services Involved:

  • Acceptor → AssocServant → DICOM Receive
  • RCV3DProcessor → Database
  • CDAR Archive Integration → FileCopy

Places to Look:

  • VIMS Monitor (check processing status)
  • System Health Study Queue
  • File system (verify space on C:, DICOM Receive, NAS drives)
  • Delete and resend test data
  • Run Wireshark to analyze DICOM communication

Critical Log Files:

  • DICOM Receive: C:\ProgramData\Vitrea\Logs\VIMS\DicomReceive.log
  • Study Tracker: C:\ProgramData\Vitrea\Logs\AvCore\StudyTracker\
  • VSP Management: C:\ProgramData\Vitrea\Logs\Management\vitrea-management.log
  • CDAR Integration: C:\ProgramData\Vitrea\Logs\VIMS\DicomReceive.CDARIntegration\
Reference: For a full list of services, see KB 7998.

Date/Time Conversion Utility

Applies to: Enterprise, Enterprise Single Server, Vitrea Extend, Vitrea Workstation deployments

This scalar function converts DICOM date/time strings to datetime2:

IF EXISTS (SELECT * FROM sys.objects WHERE object_id = OBJECT_ID(N'[Dicom].[udf_StringToDateTime2]') AND type in (N'FN', N'IF', N'TF', N'FS', N'FT'))
DROP FUNCTION [Dicom].[udf_StringToDateTime2]
GO

CREATE FUNCTION [Dicom].[udf_StringToDateTime2]
(
    @datestring nvarchar(10),
    @timestring nvarchar(12)
)
RETURNS datetime2
AS
BEGIN
    DECLARE @datedate datetime2;
    
    IF @datestring IS NULL AND @timestring IS NULL
        SET @datedate = SYSDATETIME()
    ELSE
    BEGIN
        -- pad the timestring to correct format: HHmmss
        SET @timestring = ISNULL(LEFT(@timestring + REPLICATE('0',10), 6), NULL)
        
        DECLARE @year nvarchar(4) = SUBSTRING(@datestring, 1, 4)
        DECLARE @month nvarchar(2) = SUBSTRING(@datestring, 5, 2)
        DECLARE @day nvarchar(2) = SUBSTRING(@datestring, 7, 2)
        
        -- Validate timestring format
        IF((@year IS NULL OR @year = '')
            OR (@month IS NULL OR @month = '')
            OR (@day IS NULL OR @day = '')
            OR (CHARINDEX(':', @timestring) != 0)
            OR (CHARINDEX('-', @timestring) != 0)
            OR( @timestring > '240000'))
            SET @datedate = SYSDATETIME()
        ELSE
            SET @datedate = CONVERT(datetime2, 
                stuff(stuff(stuff(LEFT(@datestring + @timestring, 14), 9, 0, ' '), 12, 0, ':'), 15, 0, ':'))
    END
    RETURN @datedate;
END;
GO

Validate Study Date/Time:

Check valid StudyDate and StudyTime for each study in Dicom.studytable:

DECLARE
    @StudyInstanceUid nvarchar(64),
    @PatientName nvarchar(256),
    @PatientId nvarchar(64),
    @AccessionNumber nvarchar(64),
    @StudyDate nvarchar(12),
    @StudyTime nvarchar(16),
    @TempVar datetime2

DECLARE study_cursor CURSOR
FOR SELECT
    StudyInstanceUid,
    PatientName,
    PatientID,
    AccessionNumber,
    StudyDate,
    StudyTime
FROM Dicom.studytable

OPEN study_cursor;
FETCH NEXT FROM study_cursor INTO
    @StudyInstanceUid,
    @PatientName,
    @PatientID,
    @AccessionNumber,
    @StudyDate,
    @StudyTime;

WHILE @@FETCH_STATUS = 0
BEGIN
    BEGIN TRY
        SET @TempVar = [Dicom].[udf_StringToDateTime2] (@StudyDate, @StudyTime)
        --PRINT 'Valid Studies: ' + @StudyInstanceUid + ';' + @PatientId + ';' + @AccessionNumber + ';' + @PatientName + ';' + @StudyDate + ',' + @StudyTime
    END TRY
    BEGIN CATCH
        PRINT 'Invalid Studies: ' + @StudyInstanceUid + ';' + @PatientId + ';' + @AccessionNumber + ';' + @PatientName + ';' + @StudyDate + ',' + @StudyTime
    END CATCH
    FETCH NEXT FROM study_cursor INTO
        @StudyInstanceUid,
        @PatientName,
        @PatientID,
        @AccessionNumber,
        @StudyDate,
        @StudyTime;
END;

CLOSE study_cursor;
DEALLOCATE study_cursor;

  • Apply group policy and re-login

CONFIG REQUIRED Button Displays:

Applies to: Enterprise, Enterprise Single Server, Vitrea Extend, Vitrea Workstation deployments

  • Run the System Setup Script for Windows 7 from the Install the Vitrea software chapter

Credentials Screen Always Displays:

Applies to: Management Server, Application Server, Multi Modality Viewer Server (Enterprise deployment)

  • Complete the "Configure Site to Zone Assignment List" procedure in the Enterprise Deployment Active Directory Planning Guide
  • If first login, cancel the prompt, close Vitrea (AppShell), and re-launch

Database Deployment Tool Failed:

Applies to: Enterprise Single Server, Vitrea Extend, Vitrea Workstation deployments

  1. Click EXIT on the Vitrea installer screen
  2. Run services.msc and restart SQL Server (VITALSQLEXPRESS)
  3. Open SQL Server Management Studio, set VIDicom to MULTI_USER
  4. Run C:\ProgramFiles\Vital Images\DatabaseDeploymentTool\ddt-ui.exe
  5. Deploy VSP, Dicom Servers, and Vitrea Workstation schemas
  6. Apply permissions and reboot

Error Creating FMS Artifact for Report Page:

Applies to: Enterprise deployment

  1. Open gpedit on Management Server and Application Servers
  2. Navigate to Computer Configuration | Administrative Templates | System | Internet Communication Management | Internet Communication Settings
  3. Enable "Turn off Automatic Root Certificates Update"

Fails to Create Evidence:

Applies to: Management Server (Enterprise deployment)

  • Ensure Management Server has full NAS access
  • Set IIS max content length to 4294967295

HTTP 403 Error During Launch:

Applies to: Enterprise, Enterprise Single Server, Vitrea Extend, Vitrea Workstation deployments

  1. Open Region settings, select Administrative language settings
  2. Ensure Welcome screen settings list English (United States)
  3. Copy settings to Welcome screen and system accounts, then reboot

Licensing Error for Cedars:

Applies to: Enterprise deployment

  • Ensure Application Server can reach Floating License Server on port 6433

RD Session Host Licensing Mode Not Configured:

Applies to: Management Server (Enterprise deployment)

  1. In PowerShell, run: $obj = gwmi -namespace “Root/CIMV2/TerminalServices” Win32_TerminalServiceSetting; $obj.SetSpecifiedLicenseServerList(“licserver.domain.local”)
  2. In regedit, set HKLM\SYSTEM\CurrentControlSet\Control\Terminal Server\RCM\LicensingMode to DWORD 2 (Per Device) or 4 (Per User)
  3. Restart the Management Server

Applies to: Enterprise, Enterprise Single Server, Vitrea Extend deployments

  • Check IIS configuration to ensure the Report Editor service is running.
  • Verify the Report Editor URL is correctly configured in System Settings.

    wmic service where "name like 'Vitrea.%%'" stopservice

Version 7.15.5: Application Service Not Checking into SQL

Applies to: Enterprise, Enterprise Single Server, Vitrea Extend deployments

  • Change .NET app pools in IIS from 2.0 to 4.0:
    1. Open IIS Manager
    2. Navigate to Application Pools
    3. Select the Vitrea application pool
    4. Change .NET CLR Version to v4.0
    5. Restart the application pool

Version 7.15.6: Web Services Down, Exporting Fails

Applies to: Enterprise, Enterprise Single Server, Vitrea Extend deployments

  • Issue: Web services are down, and exporting fails immediately with an error (no Wireshark activity).
  • Cause: IIS Restriction script was run, and the IP address changed.
  • Fix:
    1. Open IIS Manager
    2. Navigate to the Vitrea site → IP Address and Domain Restrictions
    3. Remove the old IP address
    4. Run iisreset
    5. Add the new IP address if needed
