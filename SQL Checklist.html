1. Role Definitions

Database Administrator

Sets up and manages databases and database security. Ensures standards are met during remote SQL installation. Troubleshoots database issues.

Domain Administrator

Manages Active Directory users and groups, adds/removes devices from domain. May provide static IP addresses and hostnames.

IT Administrator

Deploys projects, administrates local workstations/servers, manages applications, performs general setup and troubleshooting.

Network Administrator

Manages network connections, provides IP addresses and DNS settings, troubleshoots network issues.

VMware Administrator

Manages VMware infrastructure, creates guest instances, manages NVIDIA license server for vGPU/vDGA.

4. Configure SQL Server

Path: Start > Microsoft SQL Server [version] > SQL Server [version] Configuration Manager

  1. Expand SQL Server Network Configuration
  2. Select Protocols for VITREASQLSTD
  3. Right-click Shared Memory and select Disable
  4. Verify Named Pipes is disabled
  5. Double-click TCP/IP and select IP Addresses tab
  6. Scroll to IPAll section
  7. Clear TCP Dynamic Ports field
  8. Set TCP Port to 2222 (or customer-defined port)
  1. Select SQL Server Services in left pane
  2. Right-click SQL Server (VITREASQLSTD)
  3. Select Restart

Service should be in Running state

6. Post-Installation Configuration

Download from: SQL Server Management Studio download

To verify SPN is registered:

setspn -q MSSQLSvc/[ServerName].[Domain]:2222
setspn -q MSSQLSvc/[ServerName].[Domain]:VIMSSQLEXPRESS

To register SPN (run by domain admin):

setspn -S MSSQLSvc/[ServerName].[Domain]:2222 [ServiceAccount]
setspn -S MSSQLSvc/[ServerName].[Domain]:VIMSSQLEXPRESS [ServiceAccount]

Example:

setspn -S MSSQLSvc/CODVMLAPPVT701.hea.corpad.net:2222 HCA\CODVSvcVITSQLTest
  1. Open SQL Server Management Studio
  2. Right-click instance > Properties
  3. Select Memory
  4. Set Maximum server memory (in MB) to maximum of 4096 MB (4GB)
  5. Click OK
  1. Run services.msc
  2. Right-click SQL Server CEIP > Properties
  3. Set Startup type to Disabled
  4. From Start menu: Programs > SQL Server 20xx Error and Reporting
  5. Check both options:
    • Send Windows Error and SQL Error Reports
    • Send feature usage reports

Run in SSMS:

USE master;

8. Troubleshooting

To set AES as default encryption method:

  1. Open Active Directory Users and Computers
  2. Navigate to Service Accounts folder
  3. Double-click SQL Service account
  4. Select Attribute Editor tab
  5. Double-click msDS-SupportedEncryptionTypes
  6. Set value to 28 (enables RC4, AES128, AES256)
  7. Or 24 for AES128 and AES256 only

Symptoms: "Page failed to load" with error UNKNOWN (-338)

Solution: Verify/register SPN as shown in section 6.2
