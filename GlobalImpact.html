SingHealth

Advanced Visualization Cybersecurity Modernization

Background & Challenge

In response to a 300% surge in ransomware attacks targeting healthcare imaging systems, SingHealth initiated a strategic project to eliminate a critical vulnerability: the reliance on Remote Desktop Protocol (RDP) as the primary entry point for its advanced visualization platform.

The existing Vitrea AV solution had reached end of life/End of support. The challenge was to architect a solution that adhered to stringent Zero Trust (NIST SP 800-207) and accessibility (WCAG 2.1) standards without disrupting clinical workflows across its three-hospital network (SGH/NCCS/NHCS, CGH, and KKH).

Solution: A Re-architected, Zero-Trust Platform

The project involved a phased transition from a legacy, RDP-dependent Vitrea v6.7.4 system to a modernized, HTML5 zero-footprint architecture (Vitrea Advanced v7.14.2). This re-architected solution was specifically designed to meet SingHealth's cybersecurity requirements and features a meticulously segmented infrastructure:

  • Elimination of RDP: All client delivery is now conducted through encrypted HTTPS/TLS 1.3 sessions via HTML5, completely removing exposed RDP ports.
  • Segmented & Secure Architecture: Core components are deployed on separate VLANs to minimize lateral movement attack vectors.
  • Key enhancements include:
    • Upgrade current Vitrea AV licenses to latest version
    • Consolidation of all applications uniformly across all of 35 SingHealth concurrent licenses
    • Deploy new additional clinical solutions as part of the enterprise deployment
    • Integrate with multiple "best in breed" thin slice archiving solutions and existing PACS, into single AV deployment across the SingHealth enterprise
    • Adhere to the evolving cybersecurity guidelines (i.e., the Solution and hardware refresh should be in line with Synapxe agreed cybersecure Vitrea architecture)
  • Separation of Management Functions: The central Management Server is decomposed into dedicated servers for software hosting, SQL database operations, and RDP brokerage, ensuring a security breach in one component does not compromise others.
  • Secure Integration: The system integrates seamlessly with the existing 'Thin Slice Life Cycle Management system' and PACS for image retrieval and permanent archiving, maintaining context switching capabilities familiar to clinicians.
  • Centralized Management: A centralized monitoring and logging solution provides enterprise-wide visibility for streamlined troubleshooting and security oversight.

The primary obstacle is the ESS appliance itself. It is an all-in-one solution that cannot:

The only viable path to true redundancy is to migrate from the ESS to a full Vitrea Enterprise Suite (VES) architecture. The proposed solution involves creating two identical, synchronized datacenters.

  • Upgrade Path: Convert the existing ESS hardware into a 6-User Application Server.
  • Primary & Secondary Datacenters: Each datacenter will contain:
    • 1x Virtualized Management Server: Hosts the core database and services. Virtualization is recommended for cost savings and easier recovery.
    • 1x 6-User Application Server: Processes the advanced visualization workload. (The existing hardware is repurposed for one of these).
