Quick Tips and Summary

Key Troubleshooting Principles

  • Deployment Specificity is Key: Always note which deployment type (Enterprise, Enterprise Single Server, Vitrea extend, or Vitrea workstation) is affected as troubleshooting steps often vary.
  • Check Log Files First: Logs are your primary source of detailed error information.
  • Verify Permissions: Many issues stem from incorrect NASROOT permissions or service account permissions.
  • Restart Services Judiciously: Restarting relevant Vitrea services is a common resolution.
  • Localhost Access for Administration: When accessing Vitrea Applications and administration screens via URL for troubleshooting, always do so from the localhost.

Common Component Issues

  • Application Server: Prone to "All sessions in use" errors, resource acquisition failures (GPU related)
  • Management Server: Issues include database deployment errors, FMS artifact errors, SSO lockout
  • Vitrea Client (AppShell): Common issues include not launching, timeout errors, connectivity problems
  • Database (VIDicom, SQL Server): Errors range from failed deployment tool operations to connectivity issues

Important Ports

  • Port 443 (HTTPS): For PACS Integrations
  • Port 3002: DICOM Receive
  • Port 3003: Query Retrieve
  • Port 3389: Remote Desktop Protocol
  • Port 6433: Floating License Server for Cedars software
  • Port 2222: Recommended TCP port for SQL Server inbound connections

Common Log Locations

  • AppShell: C:\ProgramData\Vitrea\Logs\AppShell
  • Cedars QBS and QGS/QPS: C:\ProgramData\Vitrea\Logs\CSMCCardiacSuite
  • Performance Logs: C:\NAS\logCollection\current\{year}/{month}\{date-guid}
