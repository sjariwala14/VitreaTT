Cybersecurity Shared Responsibility

Proper security measures, such as security management for the operation of the product and guidelines for connecting the product to the network or peripheral devices, are required to protect against cybersecurity attacks.

Attacks come in the form of bad actors and/or malicious software (malware) such as computer viruses or worms. The risks due to cyberattacks include data falsification, leakage of information (including patient data), product malfunction, and infection of other products/devices.

Shared Responsibility Model

Security is a shared responsibility between Canon Medical and the customer. The customer is responsible for the physical infrastructure, including hardware and network, as well as the operating environment of the software.

Customer Responsibilities
  • Physical infrastructure and network
  • Operating system management
  • User access controls
  • System updates and patches
  • Security monitoring and logging
  • Storage encryption
  • Disaster recovery
Canon Medical Responsibilities
  • Software security features
  • TLS encryption for communications
  • Authentication and authorization integration
  • Security updates and patches
  • Vulnerability management
  • Security testing

Security Controls

For Customers

The hardware, network and Windows OS provides security controls for the customer to be aware of and maintain:

  • Hardware and port restrictions: Customers are responsible for any physical safeguards needed to meet their policies.
  • Identity and access management: Users and their authorization are configured through standard Microsoft controls.
  • HIPAA and GDPR requirements: Customers are responsible for limiting access to patient data to authorized individuals.
  • System updates: The customer is responsible for ensuring system updates are applied in a timely manner.
  • Security monitoring and logging: This is done using the operating system and other system controls.
  • Threat detection and forensics: This is handled by the Windows operating system and other windows tools.
  • Anti-virus and malware: Vitrea AV integrates with Windows Defender anti-virus and malware tools.
  • Storage/encryption at rest: Best practices are to encrypt all storage systems.
For Canon Medical
  • TLS encryption: Vitrea AV is configured to use TLS to protect communications channels.
  • Authentication control: Product user roles are predefined and incorporate the principle of least privileges.
  • Security updates: Customers on a current maintenance contract may be notified when new versions are available.
  • Security bulletins: Provided for customers through Canon Medical support channels.
  • Vulnerability reporting: Canon Medical should be immediately contacted in the event of a security incident.
