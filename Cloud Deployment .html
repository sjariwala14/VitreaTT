Cloud Deployment Overview

Vitrea Advanced Visualization is compatible with cloud environments. Canon Medical extends Vitrea Advanced Visualization (AV) cloud-compatibility for cloud hosted environments.

Canon Medical has chosen the most optimal configuration for typical cloud environments, but other options may be suitable depending on business use cases.

Key Factors to Consider

  • Network topology: Design impacts performance and reliability
  • Distance from data centers: Affects latency and throughput
  • Storage and compute compatibility: Not all technologies are supported in clinical systems
  • Data volume: Impacts storage requirements and processing power

Canon Medical will work with you to explore the best configuration for your situation, given the complexities involved in operating clinical systems.

Deployment Options

Single Server
Multi-Node

Single Server Deployment

Recommended for smaller implementations with fewer concurrent users (1-4 users).

Key characteristics:

  • All components on a single virtual machine
  • Simpler to deploy and manage
  • Cost-effective for small workloads
  • Limited scalability

Multi-Node Deployment

Recommended for larger implementations requiring scalability and high availability.

Key characteristics:

  • Separate management and application servers
  • Horizontal scaling of application servers
  • Better fault tolerance
  • More complex to configure
  • Higher initial cost but better for growth

Remote Access Client Specifications

Minimum Requirements

Processor: 1 GHz 64-bit (x64) with 2 or more cores
RAM: 4 GB
Storage: 40 GB (Win10) / 64 GB (Win11) with at least 1 GB free
Browser: Chrome, Edge, or Firefox for PACS integrations
Resolutions:
  • Minimum: 1280×1024
  • Recommended: 1920×1200
  • Maximum: 3840×2160 (4k)
Network: 1.0 Gbps connection
Bandwidth: Minimum 5Mbps with 100ms latency

Supported Operating Systems

  • Microsoft Windows 10 Professional 64-bit edition
  • Microsoft Windows 11 Professional 64-bit edition (Recommended)
  • Macintosh client access support – Microsoft remote desktop application 8.0.32 or later for Mac OS X 10.9

End of life notice: Windows 10 support from Microsoft will end on October 14th, 2025. Canon Medical recommends upgrading to a newer version of Windows for future compatibility, supportability, and security.

Network Topology Considerations

For optimal performance, consider these network factors:

  • Place client workstations as close as possible to the cloud region being used
  • Ensure adequate bandwidth for the number of concurrent users
  • VPN connections should have low latency (<100ms) for responsive interaction
  • For DICOM transfers, ensure sufficient sustained throughput

Additional Configuration Considerations

General Requirements

  • CPU Instructions: All systems must support AWX2 or newer CPU instructions
  • Data Throughput: Network performance between storage and cluster nodes impacts data loading speeds
  • Management Node: CPU speed affects data ingestion rates due to DICOM compression handling

Instance Reservations

While any reservation type can be used for the VMs running AV, we recommend:

  • Avoid spot instances - they can be stopped by the cloud provider at any time
  • Use reserved or on-demand instances for production workloads
  • With on-demand, verify instance availability especially if stopping them during off hours
  • Reserved instances can provide cost savings for predictable workloads

GPU Configuration

  • GPU instances must be configured in WDDM mode for Vitrea AV
  • NVIDIA licensing is handled by the cloud provider
  • Different cloud providers have specific driver requirements:
    • Azure requires Azure-specific NVIDIA drivers
    • AWS and GCP can use standard NVIDIA drivers

Storage Considerations

  • Choose storage tiers based on performance requirements
  • Higher IOPS improve responsiveness with large datasets
  • Consider separate disks for OS, applications, and patient data
  • Ensure adequate capacity for:
    • Current patient data
    • Projected growth
    • Temporary processing space

Network Topology

Design your network architecture considering:

  • Latency between components
  • Bandwidth requirements for:
    • DICOM transfers
    • Remote desktop protocols
    • Inter-node communication
  • Security requirements for healthcare data
  • Integration with existing PACS and modalities
