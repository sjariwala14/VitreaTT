Vitrea Multiple Monitor Setup Checklist

0%

This checklist provides step-by-step instructions for configuring a multiple monitor setup for Vitrea Advanced Visualization software, as outlined in the "Vitrea Advanced Visualization Multiple Monitor Setup Guide" (VPMC-17222E). Complete all steps to ensure a successful multi-monitor deployment.

Pre-Setup Requirements

Verify Existing Vitrea Deployment Page 8

Confirm a fully functional Vitrea Enterprise, Vitrea Enterprise Single Server, Extend, or Workstation deployment is installed and configured.

Action: Log in to Vitrea and verify that the system loads the Study List and applications without errors.

Check Monitor Compatibility Page 10, 13

Ensure all connected monitors have identical resolution, DPI, and orientation settings if Vitrea spans multiple monitors.

For single-monitor Vitrea display:

  • If using a monitor with a scale factor >100%, set it as the primary monitor in Windows.
  • If using 100% scaling, any connected monitor can display Vitrea without being the primary monitor.

Supported resolutions: Minimum 1280x1024, Recommended 1920x1200, Maximum 2560x1600.

Action: In Windows Display Settings, verify that all monitors share the same resolution, DPI, and orientation (e.g., all set to 1920x1200, 100% scaling, landscape).

Multiple Monitor Configuration Setup

Launch Vitrea Software Page 10

Log into the Vitrea system.

For Vitrea Thin Client, establish an RDP connection.

Action: Ensure a stable RDP connection for Thin Client users and verify login credentials work correctly.

Access Monitor Preference Settings Page 10-11

Select the gear icon in the upper right corner of the Vitrea software.

Choose "Set Monitor Preference" and click OK to enable monitor selection on the next launch.

Action: Confirm the gear icon is accessible and the "Set Monitor Preference" option appears in the menu.

Configure Display Selection Page 11

Relaunch Vitrea software on a system with multiple monitors.

On the "Display Selection - Step 1 of 2" screen, choose between:

  • "One Display" for single-monitor use.
  • "All Displays" for multi-monitor configuration.

Click the "All Displays" button to enable Vitrea to span multiple monitors.

Action: Move the Display Selection window to the desired main monitor and click "Continue" to confirm the selection.

Multiple Monitor Workflow Configuration

Set Up General Multi-Monitor Workflow Page 14-15

Open the Study List from the Applications Home screen; it displays on the main monitor.

Launch a clinical application, which will display on the next available monitor.

Optionally, drag the Study List or clinical application to any desired monitor.

Verify ability to preview a series in the Study Tab on one monitor while a clinical application runs on another.

Note: If previewing a different patient, the application monitor may gray out; click the application monitor to restore.
Action: Test dragging applications between monitors and confirm the Study List and clinical applications display as expected.

Set Up Mirada XD Multi-Monitor Workflow Page 15

Open the Study List and select the Mirada XD application from the Applications Home screen.

From the Mirada XD application, select "Window" > "New Window" to launch a second window on the secondary monitor.

Verify that actions in the secondary monitor are linked to the primary monitor.

Test closing and relaunching primary or secondary windows using "Window" > "New Window."

Action: Confirm the Mirada XD application opens a second window on the secondary monitor and that interactions are synchronized.

Handle Monitor Configuration Changes Page 14

Confirm that if a monitor is added or removed, Vitrea reverts to single-monitor mode, moving all open tabs to the remaining monitor.

For Vitrea Thin Client or Workstation, verify that minimizing the AppShell window minimizes it across all monitors.

Action: Simulate adding or removing a monitor (e.g., disconnecting a monitor) and verify that Vitrea reverts to single-monitor mode correctly.

Troubleshooting Scenarios

Issue 1: Vitrea Fails to Detect All Connected Monitors

Cause: Windows may not recognize all monitors, or monitor settings (resolution, DPI, orientation) are mismatched.

Action:

  • Verify all monitors are detected in Windows Display Settings.
  • Ensure identical resolution, DPI, and orientation settings across all monitors (Page 10, 13).
  • Restart Vitrea after adjusting Windows settings.
  • If issue persists, reconnect monitors and restart the system.

Issue 2: "Display Selection" Screen Does Not Appear on Relaunch

Cause: Monitor preference settings may not have saved correctly, or Vitrea is not in multi-monitor mode.

Action:

  • Re-access "Set Monitor Preference" via the gear icon and click OK (Page 11).
  • Ensure multiple monitors are connected and powered on before launching Vitrea.
  • Check Windows Display Settings to confirm monitors are set to "Extend" mode, not "Duplicate."

Issue 3: Application or Study List Does Not Display on Desired Monitor

Cause: Incorrect monitor selection during setup or scaling issues.

Action:

  • Repeat the "Configure Display Selection" step, ensuring the Display Selection window is moved to the desired main monitor (Page 11-12).
  • Verify the target monitor's scaling is set to 100% or set as the primary monitor if >100% (Page 10).
  • Manually drag the application or Study List to the desired monitor and test functionality.

Issue 4: Application Monitor Grays Out When Previewing a Different Patient

Cause: Expected behavior when previewing a different patient in the Study List (Page 15).

Action:

  • Click within the grayed-out application monitor to restore the view (Page 15).
  • Ensure the correct patient is selected in the Study List to avoid conflicts.

Issue 5: Mirada XD Second Window Does Not Open on Secondary Monitor

Cause: Incorrect monitor configuration or Mirada XD settings issue.

Action:

  • Confirm "All Displays" was selected during the Display Selection step (Page 11).
  • Re-select "Window" > "New Window" in Mirada XD (Page 15).
  • Verify Windows recognizes the secondary monitor as an extended display.
  • If issue persists, restart the Mirada XD application and reattempt.

Issue 6: Vitrea Reverts to Single-Monitor Mode Unexpectedly

Cause: Monitor configuration change (e.g., disconnected cable, sleep mode) detected by Vitrea (Page 14).

Action:

  • Check all monitor connections and power status.
  • Re-run the "Set up multiple monitor configuration" procedure (Page 10-12).
  • Avoid changing monitor configurations while Vitrea is running.

Issue 7: Performance Issues or Lag in Multi-Monitor Mode

Cause: Insufficient system resources or outdated graphics drivers.

Action:

  • Verify system meets Vitrea's hardware requirements (refer to the Vitrea Installation Guide, Page 5).
  • Update graphics drivers to the latest version compatible with your system.
  • Reduce the number of open applications or tabs in Vitrea to lower resource usage.
  • Contact Canon Medical Support if performance issues persist (Page 5).

Additional Notes

Understand Display Regions Page 10

Confirm display regions are numbered from left to right, top to bottom (e.g., 1, 2, 3, 4 for a 2x2 layout).

Ensure physical monitors split into multiple regions are recognized as separate displays by Windows.

Action: In Windows Display Settings, verify monitor numbering matches the expected layout.

Validate Application Behavior Page 14

For Vitrea Thin Client: Confirm Mirada applications on the secondary monitor are not linked to the primary monitor's patient application.

For Mirada XD: Only this application can span multiple monitors; verify functionality.

Action: Test Mirada XD's multi-monitor functionality separately from other applications.
