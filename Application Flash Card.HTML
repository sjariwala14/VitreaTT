Troubleshooting Scenarios (25)

1. The Vitrea software installation fails, and you suspect a Group Policy is preventing the prep script from running successfully.

Solution: Manually run the Prep Script. Navigate to E:\Vitrea_Resources\<media contents>\Utilities\OS Configuration Scripts\<your setup OS version>\Prep. Open PowerShell as administrator, cd to that path, and then type .\runscript.bat. If a security warning displays, type R and press Enter. Select the appropriate component (e.g., Application Server). After execution, type A for "Yes to All" to reboot the system. Then, create the SkipSysPrep registry key (HKLM/Software/Vital Images/Installer, String Value, SkipSysPrep=1) and reboot. After successful Vitrea installation, remove this key.

2. You are trying to install the Vitrea software, but you are signed in using the built-in Administrator account, and the installation fails because the Username field does not authenticate.

Solution: Ensure you are signed in using a domain account with local administrator rights (e.g., Vitrea-Admin). The built-in Administrator account will not authenticate as it is a local administrator, not a domain administrator.

3. During the Vitrea installation, a message displays stating, "There is a pending reboot. Please reboot the system now otherwise installation may not succeed."

Solution: Canon Medical recommends you click the Exit button and reboot the system before attempting the installation again.

4. The Vitrea software installation process is showing Command Prompt screens, and you are unsure if you should interact with them.

Solution: Do not interact with the Command Prompt screens. They will close automatically when the required processing is complete.

5. After entering SQL Server connection details, the verification fails when you click the TEST button.

Solution: Click the BACK button to return to the Configure the database connection screen and re-enter the hostname, instance name (if applicable), port, and database name (steps 6-10).

6. You need to confirm that your Data Center GPU is compliant and configured for vGPU in a virtualized Application Server, but you are receiving an error.

Solution: Refer to the provided location (implied external document) to determine if your card is compliant. The error displays because the vGPU requires Tesla Graphics cards to be configured in Windows Display Driver Model (WDDM) mode. Consult the "Setting TCC Mode for Tesla Products" article for more information.

7. The performance of your Vitrea applications using NVIDIA vGPU is degrading over time, and you suspect a licensing issue.

Solution: Verify the NVIDIA vGPU licensing. Check the NVIDIA Control Panel (Manage License task) to ensure the Primary License Server field contains the correct hostname. If it's missing, contact the site's VMware administrator. Remember that performance degrades (e.g., frame rate capped at 15 fps after 20 minutes, CUDA stops working after 24 hours) if a license is not obtained.

8. You need to enable SSL/TLS for your Vitrea deployment, but your PowerShell ExecutionPolicy is set to Restricted.

Solution: Before running the ConfigureTLS_Enterprise.ps1 script, set the ExecutionPolicy to Unrestricted by typing Set-ExecutionPolicy UnRestricted and pressing Enter. After completing the TLS configuration procedure, remember to set the ExecutionPolicy back to Restricted.

9. You are attempting to run the ConfigureTLS_Enterprise.ps1 script, and you are using a certificate thumbprint that contains spaces.

Solution: Ensure the thumbprint hash you use does not contain any spaces in its value. Update the hash to remove spaces (e.g., bdea4c20896eb0a9adce9c97314d11887782c330 instead of bd ea 4c 20 ...).

10. You are unable to open the Certificate Store under the Local Computer Certificate Store as required for SSL/TLS configuration.

Solution: Ensure you are explicitly opening the Local Computer Certificate Store by running certlm.msc from the Start menu.

11. The Vitrea software displays a blank gray screen on startup, and the Session Helper log shows "Error: unable to get issuer certificate" when TLS is enabled.

Solution: This can occur if the system has more than one digital certificate trust chain from the CA and one is expired. Log into all Management and Application Servers as an administrator. Access the computer certificate stores (certmgr.msc). Expand "Trusted Root Certification Authorities" and select "Certificates". Remove any expired certificates for the issuing CA, but do not remove valid ones. Restart any server where certificates were removed. If the certificate is restored, contact the customer's Active Directory Administrator.

12. You are performing administrative work on a Vitrea system, but the Vitrea application is running in kiosk mode, preventing access to the Windows desktop.

Solution: From within the Vitrea software, select the double-gear icon in the upper right corner, then select the "Reveal Desktop" option. This launches Windows Explorer and reveals the Start menu. The system will reset to defaults when the Kiosk session is closed.

13. You need to test IIS settings configured to restrict IP addresses on an Application Server after setting them up on the Management Server.

Solution: Copy the TestIIS.ps1 script from the Management Server to the Application Server. Open PowerShell as Administrator, cd to the script's location, and type .\TestIIS.ps1 -ManagementIp <IP address of the Management Server> and press Enter. Verify that IIS endpoints are allowed and there are no unexpected failures.

14. You have deployed an Application Server and need to configure matrix filter support for Just-In-Time (JIT) processing.

Solution: Connect to the Management Server and open Windows PowerShell with Administrator rights. Navigate to E:\Vital_Resources\Vitrea-[version number]\Utilities\Scripts\ConfigureJiT. Enter the command: .\LimitJitTo512Data.ps1 -ApplyToAllExcept TAVRAdvisor,CTColonCADJIT,CaScoreAdvisor,LungCADAdvisor,LungDensityAdvisor,CTLungTextureAnalysisJIT,CTBoneRemovalJIT,CarotidAutoVesselJIT and press Enter.

15. You encounter an "The application is currently unlicensed" error when trying to launch Mirada XD Administration.

Solution: Navigate to C:\Program Files\Mirada Medical\Mirada XD\bin folder. Copy the plugin, plugin.xml.hmac, and newEmbeddedWindow.pkg files to any location outside of the current one. After completing the administration tool tasks, move these files back to the original bin folder.

16. You need to update the .NET Core Runtime components due to a Microsoft Windows vulnerability announcement, and you want to ensure you download the correct versions.

Solution: Open Control Panel > Programs > Programs and Features to confirm existing runtime versions. Then, access https://dotnet.microsoft.com/en-us/download/dotnet. In the Version column, select the corresponding release version (e.g., .NET 6, .NET 8) to find newer security patches. Only install the minor version updates from the same "family". Ensure you install all three Runtime components (.NET Runtime, ASP.NET Core Runtime, .NET Desktop Runtime). Reboot the system afterwards.

17. Client Computer Local Drives are visible in the Save File screen from a remote session, and you need to disable this Thin Client Local Drive Access feature.

Solution: On the Application Server, open Group Policy Management (via Start menu search for Administrative Tools). Expand Forest > Domains > Group Policy Objects. Right-click the Vitrea Application Servers Group Policy Object and select Edit. Navigate to Computer Configuration > Policies > Administrative Templates > Windows Components > Remote Desktop Services > Remote Desktop Session Host > Drive and Resource Redirection. Double-click "Do not allow direct redirection" and select the "Enabled" radio button to disable the feature, then click OK.

18. You have purchased DEIImageGenerationPlugin 3.4.0, Spectral Analysis, Spectral Composition Analysis, and Spectral Stone Analysis (VSTP2 applications), but they were not installed during the initial software installation.

Solution: Sign in as an administrator. On the Application Server, open Command Prompt as Administrator. Navigate to the Vitrea installation folder, then the Bundle folder. From the Bundle folder, open DEImageGenerationPlugin\3.4\install and type DEImageGenerationPlugin.msi to install it. Then, type CD ..\..\.. to return to the Bundle folder and install SpectralAnalysisInstaller_x64.msi, SpectralCompositionAnalysisInstaller_x64.msi, and SpectralStoneAnalysisInstaller_x64.msi.

19. You accidentally installed VSTP2 Spectral Analysis applications without purchasing them, and you need to revert to VSTP1 Spectral Analysis applications.

Solution: Sign into the system as an administrator. On the Application Server, open Command Prompt as Administrator. Navigate to the Vitrea installation folder, then the Bundle folder. Type CMSCWSClinicalApp.msi and follow the prompts for Repair. Click Yes to restart your system when prompted.

20. After installing VSTP2 Spectral Analysis, you need to set up the Hanging Rule Input and Auto Processing Settings, but the export destination is not available.

Solution: Ensure you have completed the "Attach an Archive" procedure in the Management Server Install Guide. This is required for the export destination to be selectable in the Hanging Rule Input and Auto Processing Settings screen.

21. You are deploying virtual Application Servers, and the Vitrea software is running inefficiently with a warning in System Health about too many sockets.

Solution: When deploying virtual Application Servers, ensure you set a maximum of 1 or 2 sockets for each guest. The default is often the same number of sockets as vCPUs (e.g., 8 sockets and 8 cores), which should be changed to 1 socket and 8 cores for efficiency.

22. You plan to use Global Illumination with Vitrea, but the rendering is not working as expected.

Solution: Be aware that Global Illumination rendering requires an NVIDIA GPU. CPU-based rendering is not supported for GIR. Ensure your GPU meets the minimum requirements (e.g., CUDA10-minimum capability).

23. Vitrea Advanced Visualization is not running or performing correctly, and you suspect an issue with CPU instruction set extensions.

Solution: Run the Enumerate CPU Instructions PowerShell script to retrieve each machine's exposed extensions. Verify that the required extensions (SSE, SSE2, SSE3, SSSE3, SSE4.1, and SSE4.2) are available. If not, work with the customer site's Hypervisor administrator to ensure the Guest has the proper EVC CPU mode enabled.

24. You notice inconsistent performance and user experience in your vGPU deployment.

Solution: Ensure that the NVIDIA vGPUManager is set to "Fixed share" scheduling (recommended) or "Equal share" (acceptable). The "Best effort" scheduler (default) is not supported and can lead to inconsistent performance.

25. You need to confirm the status of security and best practice configurations set by the system hardening script after Vitrea installation.

Solution: Open the SysChkrReport {machinename,date,time}.html file in any Internet browser. This report confirms the status of security and best practice configurations, displaying "OK" in green or "Not OK" in red. If you opted out of running the script during installation, you can run VitreaSystemChecker.ps1 manually from E:\Vitrea_Resources\<media contents>\Utilities\Scripts\SystemChecker.
